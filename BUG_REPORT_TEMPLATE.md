# Bug Report Template

Copy this file (or its contents) and fill in the fields. Email the result to the address in your beta welcome note. The more you fill in, the faster we can fix it.

```
TITLE: <one-sentence summary>
SEVERITY: <P0 / P1 / P2>
DATE: <YYYY-MM-DD>
URL: <full URL where it happened>
ROLE: <buyer / seller / dealer / service_provider / admin>
EMAIL OF ACCOUNT USED:

WHAT I WAS TRYING TO DO:
<one or two sentences>

WHAT I EXPECTED TO HAPPEN:
<one sentence>

WHAT ACTUALLY HAPPENED:
<as much detail as you can give — exact error messages, button labels, sequence of clicks>

REPRODUCIBLE?: <yes / no / sometimes>

STEPS TO REPRODUCE:
1.
2.
3.

ATTACHMENTS:
- Screenshot(s)
- Browser console (F12 → Console → screenshot or copy text)
- Network tab if a request failed (Status, response body)

ENVIRONMENT:
- Browser + version (e.g., Chrome 121 / Safari 17.3 / Firefox 122)
- Device (desktop / iOS / Android)
- Screen size if mobile (rough is fine — "iPhone 14 Pro")

BLOCKER?: <yes — I can't continue / no — I worked around it>
WORKAROUND (if any):

ANY HYPOTHESIS:
<optional — your guess at root cause if you have one>
```

## Severity guide

- **P0** — site-down, payments broken, data loss, can't log in, security exposure. Tell us by text immediately, not email.
- **P1** — major flow broken (can't list, can't message, leads inbox empty when it shouldn't be).
- **P2** — minor flow broken, copy errors, visual glitches, edge-case usability.

## What's helpful even outside a bug

If something just feels weird — confusing copy, a button you didn't expect, a moment you almost bailed — that's not technically a bug, but it's gold for us. Send it under "Feedback" instead of "Bug" in the subject line.

## What we'll do with your report

- Acknowledge within 24 hours during weekdays.
- Triage to a fix-now / fix-this-week / backlog bucket.
- Tell you when it's fixed.

Thanks for filing. Every report directly improves the product before public launch.
